### 2. useNavigation.tsx (~180 lines)

**Purpose:** React Context + hook for persistent navigation state across sessions.

**Architecture:**

```
NavigationProvider (root)
  ↓
sessionStorage [STORAGE_KEY: 'gajian-aman-nav-state']
  ↓
NavigationState { activeSection, activeTabs }
  ↓
useNavigation() hook → NavigationContextValue
```

**Context API:**
- `activeSection: string` — Current primary section (read-only from path)
- `setActiveSection(sectionId)` — Update active section
- `getActiveTab(sectionId)` → string` — Get last-viewed child for section
- `setActiveTab(sectionId, childPath)` — Persist active tab per section
- `navigateToSection(sectionId)` → void` — Navigate to section's last tab

**State Persistence Strategy:**
- **Storage:** `sessionStorage` (not localStorage)
  - Rationale: Persists within browser session (tab switching, refreshes) but resets between sessions
- **Load on Mount:** `loadPersistedState()` tries to parse sessionStorage; falls back to defaults
- **Auto-sync on Change:** `useEffect` syncs state to sessionStorage after every state mutation
- **Auto-sync on Navigation:** URL changes sync back to state, updating activeTabs

**Key Behaviors:**
1. When user navigates to `/spend/budget`, context automatically:
   - Sets `activeSection = 'spend'`
   - Sets `activeTabs['spend'] = '/spend/budget'`
2. When user navigates back to Spend section later, context shows Budget tab (persisted)
3. On page refresh, sessionStorage restores exact navigation state
4. On browser session end (tab close), state is cleared (privacy-respecting)

**Error Handling:**
- Wraps `sessionStorage.getItem()` in try-catch (silently fails on quota/access errors)
- Falls back to default state if storage is corrupted
- `useNavigation()` throws if used outside `NavigationProvider` (fail-fast principle)

### 3. BottomNavigation.tsx (~160 lines)

**Purpose:** Mobile-only (lg:hidden) 5-icon bottom navigation bar with badges and active state animation.

**Component Props:**
- `badges?: Record<string, number>` — Optional badge counts per section (e.g., `{ home: 2, spend: 0 }`)

**Responsive Behavior:**
- **Mobile (<1024px):** Visible, fixed bottom-0, height 64px (16 rem = 4rem content + padding)
- **Desktop (≥1024px):** Hidden (lg:hidden class)

**Layout:**
- 5 equal-width columns (flex-1 on each item)
- Minimum 56px × 56px touch target (exceeds 44px WCAG AAA minimum)
- Stacked vertically: icon (5×5 grid, 20px) → label (10px text)

**Visual States:**

1. **Inactive:** 
   - Icon: 2px stroke, gray (`--color-content-tertiary`)
   - Label: 10px, medium weight, gray
   - No indicator

2. **Active:**
   - Icon: 2.5px stroke, Gajian green (`--color-brand-primary`)
   - Label: 10px, semibold weight, green
   - Indicator: 4px dot below icon (spring animation)

**Animations:**
- Active indicator uses `layoutId="bottomNavIndicator"` for shared layout animation
- Spring physics: `stiffness: 500, damping: 30` (bouncy, responsive)
- Respects `prefers-reduced-motion` — indicator skipped if user has animations disabled

**Badge System:**
- Position: top-right of icon (-1px, -8px offset)
- Size: 16px circle, red background (`--color-sentiment-negative`)
- Text: white, 10px bold, centered
- Overflow: Shows "99+" if count > 99
- `aria-label` announces badge count to screen readers

**Accessibility:**
- `role="navigation"`, `aria-label="Navigasi utama"` (Indonesian)
- Each link has `aria-current="page"` when active
- Each link has `aria-label` with bilingual label ("Beranda — Home")
- Focus ring: 2px outline, offset 2px, brand color
- Keyboard: Tab navigation, Enter/Space activates links
- Screen reader announces: "Beranda — Home, button, selected" (e.g., when active)

**Design Tokens Used:**
- Colors: `--color-brand-primary`, `--color-content-tertiary`, `--color-sentiment-negative`, `--color-bg-elevated`, `--color-border-neutral`
- Motion: `--duration-fast` (150ms) from Phase 02 tokens
- Spacing: Calculated pixel values (56px touch target, 4px indicator, etc.)

**Integration Points:**
- Uses `useNavigation()` hook to get `activeSection`, `getActiveTab()`, `navigateToSection()`
- Links point to `getActiveTab(section.id)` — navigates to last-viewed child, falls back to section path
- Badge counts passed as prop (Batch 2+ will wire up to actual notification counts)

## ARCHITECTURE DECISIONS

### Decision 1: sessionStorage vs. localStorage
**Choice:** sessionStorage  
**Rationale:** Persists within browser session (preserves nav state across tab switches, refreshes) but resets between sessions (privacy-respectful, prevents stale state)  
**Tradeoff:** Users returning after 24 hours start fresh (not ideal for UX but required for privacy compliance)

### Decision 2: Bilingual Labels (label + labelId)
**Choice:** Both English and Indonesian  
**Rationale:** Aligns with Principle 11 (Indonesian-First Localization); labelId is primary, label is fallback  
**Tradeoff:** Slight redundancy in config (20 extra lines) but ensures consistency with Indonesian UX

### Decision 3: Exhaustive ROUTE_REDIRECTS
**Choice:** Hardcode all 20+ old→new mappings  
**Rationale:** Preserves backward compatibility for bookmarks, browser history, external links; cost is negligible (one React render)  
**Tradeoff:** Redirect routes permanently (can't be removed without breaking existing bookmarks)

### Decision 4: Spring Animation for Active Indicator
**Choice:** Framer Motion spring physics over linear easing  
**Rationale:** Matches QPay UI Kit reference (fintech standard); feels responsive and premium  
**Tradeoff:** Slight performance cost on low-end devices (mitigated by prefers-reduced-motion support)
