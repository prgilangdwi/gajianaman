## Technical Decisions & Rationale

### ARIA Attribute Value Format
- **Pattern discovered:** ARIA attributes like `aria-pressed`, `aria-expanded` require **string values** (`'true'` / `'false'`), not boolean expressions
- **Applied to:** MobileFilterSheet wallet buttons, ExpandableTransactionRow toggle
- **Reason:** ARIA specification strictly requires string representation; boolean values fail ESLint validation

### Focus Trap Implementation (MobileFilterSheet)
- **Why necessary:** Custom modal (uses `<div role="dialog">`, not shadcn Dialog component)
- **Pattern:** `useEffect` hook queries focusable elements (`button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])`) and focuses first on open
- **Benefit:** Ensures keyboard focus stays within modal, preventing accidental focus escape to page behind

### Arrow Key Navigation (BottomNavigation)
- **Pattern:** Circular index wrapping: `(index + 1) % length` (right), `(index - 1 + length) % length` (left)
- **Benefit:** Familiar desktop app pattern; enables mobile users with external keyboards to navigate sections
- **Trade-off:** Only triggered by explicit ArrowLeft/ArrowRight (not diagonal arrows or other keys)

### Keyboard-Accessible Toggle Pattern (Wallet Buttons, ExpandableTransactionRow)
- **Pattern:** `<button type="button">` with `aria-pressed` or `aria-expanded` + `aria-label`
- **Behavior:** Native button element automatically handles Enter/Space key activation
- **No manual handler needed:** Browser default behavior is sufficient

### Focus Ring Styling
- **Applied to:** All keyboard-interactive elements (inputs, buttons, selects)
- **Syntax:** `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-[var(--color-brand-primary)]`
- **Benefit:** Clear visual feedback for keyboard users without affecting mouse users
