## Implementation Details

### TransactionModal.tsx — Edit Mode Support

**Component Enhancements:**
- Extended `TransactionModalProps` with optional `transaction` prop for edit mode
- Added `isEditMode` boolean flag: `const isEditMode = !!transaction`
- Added `useEffect` hook to populate form fields when transaction is provided:
  - Populates: amount, category, note, date, type from transaction
  - Defaults to 'manual' input method (no AI parsing in edit mode)
- Modified `handleSave()` to support both insert and update:
  - Checks `isEditMode && transaction` to determine mutation type
  - Uses `.update().eq('id', transaction.id)` for edits
  - Uses `.insert()` with user_id for new transactions
  - Shows different toast messages: "Transaksi diperbarui" (edit) vs "Transaksi ditambahkan" (new)

**Benefits:**
- Reuses existing form validation and submission logic
- No new component needed; existing modal handles both flows
- Backward compatible: existing code that adds transactions continues to work

### ExpandableTransactionRow.tsx — Action Buttons

**Component Enhancements:**
- Added `Trash2` and `PenLine` icons from lucide-react
- Added `Button` component import from shadcn/ui
- Extended `ExpandableTransactionRowProps` with optional `onEdit` and `onDelete` callbacks
- Placed action buttons in expanded details panel after AI Confidence field
- Buttons only render if callbacks provided (backward compatible)
- Edit button: Icon + "Ubah" label, calls `onEdit(tx)`

### Riwayat.tsx — Edit/Delete Integration

**State Management:**
- `isEditModalOpen`: Controls TransactionModal visibility
- `transactionToEdit`: Stores transaction being edited
- `deleteConfirmOpen`: Controls delete confirmation dialog visibility
- `transactionToDelete`: Stores transaction pending deletion
- `isDeleting`: Loading state during delete mutation

**Edit Handler (Full Implementation):**
```typescript
const handleEdit = useCallback((tx: Transaction) => {
  setTransactionToEdit(tx);
  setIsEditModalOpen(true);
}, []);

const handleEditSaved = useCallback(() => {
  setIsEditModalOpen(false);
  setTransactionToEdit(null);
  refetch();
}, [refetch]);
```
- Opens TransactionModal with transaction data
- Modal handles form population and update mutation
- On save: Closes modal, refetches transactions, shows toast
- On cancel: Closes modal without mutation

**Delete Flow:**
1. User clicks Delete button → `handleDeleteClick()` stores tx and opens confirmation dialog
2. Dialog shows transaction note/category and amount with warning
3. User confirms → `handleConfirmDelete()` executes mutation:
   - Calls `supabase.from('transactions').delete().eq('id', transactionToDelete.id)`
   - On success: Toast notification, refetch transactions, close dialog
   - On error: Toast error message (no silent failures)
4. Loading state (`isDeleting`) prevents double-clicks during mutation

**Integration Points:**
- Passes `onEdit` and `onDelete` callbacks to each `ExpandableTransactionRow`
- Leverages `refetch()` from `useTransactions` to refresh UI after delete
- Uses `AlertDialog` (shadcn/ui) for confirmation UX
