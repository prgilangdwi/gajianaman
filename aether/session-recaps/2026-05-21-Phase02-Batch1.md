### Batch 1: CSS Token Architecture (theme.css)

**Fix 1 — Dark mode legacy token normalization**
- Replaced all `oklch()` color values in `.dark` block with brand-aligned hex values
- Mapped legacy tokens to semantic equivalents (e.g., `--background: oklch(0.145 0 0)` → `#0F1110` to match `--color-bg-screen`)

**Fix 2 — Typography base layer alignment**
- Updated `@layer base` to use custom semantic tokens instead of Tailwind defaults
- Changed: `h1 var(--text-2xl)` → `var(--text-h1)`, `h2 var(--text-xl)` → `var(--text-h2)`, etc.
- Changed: `h4 var(--text-base)` → `var(--text-body-lg)`
- Result: Typography hierarchy now respects design token values, not Tailwind's generic scale

**Fix 3 — Shadow system completion (6-level)**
- Added missing shadow tokens: `--shadow-none`, `--shadow-float`, `--shadow-dropdown`
- Wired all 6 shadows into `@theme inline` block for Tailwind utility generation
- Specifications:
  - `--shadow-none: none` (no elevation)
  - `--shadow-card`: 1px lift (resting card)
  - `--shadow-card-hover`: 4px lift (interactive hover)
  - `--shadow-float`: 8px lift (FAB, floating actions)
  - `--shadow-dropdown`: 4px lift (popover, dropdown)
  - `--shadow-modal`: 20px lift (modal, high-elevation)
  - `--shadow-nav`: negative 1px (top nav separator)
- Result: Complete elevation system ready for component layering

**Fix 4 — Category color conflict resolution**
- Light mode: `--color-cat-groceries #4AE54A` (brand primary conflict) → `#10B981` (distinct green)
- Light mode: `--color-cat-income #2F5711` (sentiment-positive conflict) → `#059669` (distinct emerald)
- Dark mode: Added category color overrides for all 10 categories (previously missing)
- Result: All category colors now visually distinct; dark mode category colors match intent
