## 4. CRITICAL FINDING — Scoring Algorithm Behavior

**The react-doctor scoring formula is NOT linear with instance count.**

Despite reducing total issues from 806 to ~555 (31% reduction) and virtually eliminating two high-count rules, the score held at 60/100.

**Evidence:**
- `design-no-redundant-size-axes`: 242 → 11 violations (95% reduction) = 0 score change
- `design-no-three-period-ellipsis`: 20 → 0 violations (100% elimination) = 0 score change
- Share URL parameters before/after: `s=60&e=10&w=796&f=185` → `s=60&e=10&w=553&f=185`
- Files affected count: 185/199 — UNCHANGED despite modifications to 56+ files

**Hypothesis:** The scoring appears to be driven by:
1. **File coverage** — 185/199 files still have at least one violation (same as before, because most files we fixed still have other violations like `no-giant-component`, `design-no-bold-heading`, etc.)
2. **Permanent structural drag** — `deslop/unused-file` ×184 false positives (React.lazy limitation) + 10 errors in locked shadcn/ui files contribute heavily and cannot be fixed

**Why files affected didn't change:** We removed size-axis violations from 56 files, but those same files still have OTHER violations (accessibility, heading weight, component size, etc.). A file counts as "affected" if it has ANY violation. Since we only removed one class of violation per file, the files remain in the "affected" pool.

**Conclusion:** The ≥80 target is NOT achievable under current constraints without either:
- Eliminating the React.lazy unused-file false positives (requires react-doctor v0.2.5+ or ESLint config changes)
- Making entire files completely violation-free (requires fixing ALL violations in each file, including ones in locked files)
- Or a different scoring interpretation emerging with a future react-doctor version
