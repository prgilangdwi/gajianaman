## 3. ARCHITECTURE NOTES

- **FixedExpense `id` field**: Added `id: string` (crypto.randomUUID()) to FixedExpense type beyond the spec. This is required for React's AnimatePresence to track list items by stable key — without it, removing a middle entry would cause wrong exit animations. The GajianInput type spec did not include `id`, but it's an internal implementation detail and not part of the submit payload.
- **Wizard inline rendering**: Gajian.tsx uses `showWizard` state and conditional rendering (not routing) to show the wizard — exactly as specified. No App.tsx changes; route wiring is deferred to Batch 3.
- **Step transition direction**: Implemented forward/backward slide using AnimatePresence `custom` prop with step number as direction signal — consistent with the motion/react patterns used in the rest of the codebase.
- **Suggestion chips in Step 2**: Added quick-tap category suggestions as an enhancement that fills the first empty entry's category field, reducing friction on mobile. This is purely additive and doesn't change the validation or submit flow.
- **Submit placeholder**: `handleSubmit` only logs to console (as specified for Batch 1). Batch 2 will replace this with the `useBudgetRecommendation` hook call.
