## 3. ARCHITECTURE NOTES

- **No `id` in FixedExpense:** Spec requires the interface without an `id` field. List entries use index-based keys for React. `motion.div` + enter-only animations (no AnimatePresence exit) avoids the `ul/li` semantic issue entirely.
- **`step2Attempted` pattern:** Validates only after the user clicks "Lanjut" on Step 2. Before that, no error state is shown — avoids showing errors before the user has had a chance to interact.
- **Inline rendering (no routing):** `Gajian.tsx` uses `showWizard` local state to conditionally render the wizard in-place. App.tsx is untouched. Route wiring deferred to Batch 3.
- **Submit placeholder:** `handleSubmit` logs to console and shows a toast. Batch 2 replaces this with the `useBudgetRecommendation` hook call and navigation to `/gajian/confirm`.
- **Import pattern:** Followed spec exactly — `motion` from `motion/react`, `toast` from `sonner`, all utilities from `@/lib/utils` and `@/lib/transitions`.
