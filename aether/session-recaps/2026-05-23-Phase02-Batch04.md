## useDompetFilter Hook Specification

### API
```typescript
export function useDompetFilter() {
  return {
    selectedDompet: string | null,        // Currently selected wallet ID (null = all)
    setDompet: (walletId: string | null) => void,  // Set selected wallet
    mounted: boolean,                      // True when initialization complete (SSR-safe)
    clearDompet: () => void,              // Clear selection and use all wallets
  };
}
```

### Behavior
1. On mount, reads `localStorage['gajian_aman_dompet']`
2. If user preference exists, use it (wallet ID string)
3. Otherwise, default to null (show all wallets)
4. `setDompet()` persists choice to localStorage (or removes entry if null)
5. SSR-safe: exports `mounted` flag to prevent hydration mismatch

### localStorage Key
- **Key:** `gajian_aman_dompet`
- **Values:** Wallet ID string or omitted (null = all)
- **Example:** `localStorage.setItem('gajian_aman_dompet', '"wallet-uuid-123"')`

## Data Flow

### Income Filtering Pipeline
1. `useTransactions()` fetches all transactions (month + year filtered)
2. `useDompetFilter()` provides selected wallet ID
3. Local filter: `filteredTransactions = selectedWallet ? filter by wallet : all`
4. Local filter: `incomeTransactions = filteredTransactions.filter(t => t.type === 'income')`
5. `useWalletStats()` calculates totals and categories from income transactions
6. `useCategoryTransactions()` provides category-specific stats
7. UI renders based on data (loading, empty, error, or content)
