## useDarkMode Hook Specification

```typescript
export function useDarkMode() {
  return {
    isDarkMode: boolean,           // Current theme state
    mounted: boolean,              // True when initialization complete (SSR-safe)
    toggleDarkMode: () => void,    // Toggle theme + persist to localStorage
    setSystemPreference: () => void // Clear preference and use OS default
  };
}
```

### Behavior
1. On mount, reads `localStorage['gajian_aman_darkmode']`
2. If user preference exists ('light' or 'dark'), use it
3. Otherwise, detect system preference via `window.matchMedia('(prefers-color-scheme: dark)')`
4. Apply theme by adding/removing `.dark` class on `<html>`
5. Listen to OS preference changes only when in 'system' mode
6. `toggleDarkMode()` saves choice to localStorage as 'light' or 'dark'
7. `setSystemPreference()` clears localStorage to use OS default

## Technical Notes

### Theme Application Strategy
- `@media (prefers-color-scheme: dark)` applies to `:root` for system preference respecting
- `.dark` class on `<html>` allows manual override (necessary for themes that respect user choice)
- Both scopes define identical token values for consistency

### SSR Considerations
- Hook exports `mounted` boolean to prevent hydration mismatch
- Client-side theme detection occurs on mount (safe for SSR apps)
- No server-side theme detection needed for initial render

### Future Integration Points
- Settings page will call `toggleDarkMode()` from button click
- App wrapper (Layout.tsx) will call `useDarkMode()` on mount
- Batch 2 will add Settings UI using the hook
