## 3. ARCHITECTURE NOTES

### Error Handling Philosophy (Principle 08 Application)
- ErrorBoundary: Catches rendering errors at component tree boundary
- ScreenStates: Provide standardized UI feedback (loading/error/empty)
- Combined approach enables every page to implement "loading, error, empty states — no exceptions"
- Development mode shows error details; production shows user-friendly message

### Folder Structure Alignment with Master Roadmap
- **`components/ui/`**: Immutable shadcn/ui primitives (never edit)
- **`components/common/`**: Custom reusable components wrapping/extending ui primitives
  - ErrorBoundary (wraps child tree, provides fallback)
  - ScreenStates (composes ButtonBase, CardBase for feedback UI)
  - Future: AmountDisplay, CategoryIcon, StatusBadge, etc.
- **`components/features/`**: Feature-specific business logic (Phase 03+)

### Token System Completeness
Token mapping document serves as:
1. **Developer guide**: How to use tokens in components (CSS `var(--token-name)` syntax)
2. **Designer guide**: How to create/maintain Figma design library in sync with CSS tokens
3. **Handoff documentation**: Figma → Code workflow via Code Connect mapping
4. **Accessibility baseline**: WCAG AAA compliance verification for all color combinations
5. **Single source of truth**: All 50+ tokens documented with usage patterns

### Error Boundary Trade-offs
- **Scope:** Catches rendering errors only, not event handlers or async code
  - Rationale: Rendering errors are most critical (app-breaking); event errors are localized
- **Fallback strategy:** Generic error UI with optional custom fallback prop
  - Allows pages to customize while maintaining baseline safety net
- **Development mode:** Shows error details for debugging; hidden in production for UX
  - Follows best practice: helpful for dev, safe for users

### Screen States Design Decisions
- **Separate components** rather than single compound component
  - Benefit: pages import only what they need (code splitting friendly)
  - Flexibility: different combinations (loading → success, loading → error, empty → filled)
- **Minimum 400px height** for consistent spacing across screen sizes
  - Ensures vertical centering and breathing room on mobile/tablet/desktop
- **Indonesian-first labels** (Bahasa Indonesia)
  - Consistent with Principle 11 (Indonesian-First Localization)
  - All UI labels in Bahasa for Indonesian user base
