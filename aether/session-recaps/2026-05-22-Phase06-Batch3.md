## Technical Decisions & Rationale

### Persist Middleware Over sessionStorage
Zustand's persist middleware handles automatic serialization/deserialization. localStorage chosen over sessionStorage to preserve conversation history across browser sessions (better UX for recurring users).

### Last 10 Messages Truncation
Prevents localStorage bloat for long chat sessions. New users get empty state; returning users see recent context but not unbounded history.

### suggestedActions Type Safety
Defined `SuggestedAction` interface in store (not component) to ensure consistent shape across app. Maps API response (`suggested_actions` array) to typed actions with `id`, `label`, `prompt`.

### No State for "isUserTyping"
Kept simple: progressive disclosure uses only `isLoading` state. Full user-input-tracking (with debounce) deferred to Phase 07 if needed. Current approach prevents over-engineering.

### API Response Shape Flexibility
Handles multiple field names: `data.response || data.answer` for message content. Suggested actions are optional (`if (data.suggested_actions && Array.isArray(...))`). Graceful fallback if API schema varies.
