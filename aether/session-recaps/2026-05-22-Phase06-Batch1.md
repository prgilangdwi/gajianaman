## Key Implementation Details

### ChatBubble.tsx
- ✅ **Token-Driven Colors** (Principle 03)
  - User messages: `brand-primary` background with `brand-primary-fg` text
  - Assistant messages: `bg-card` background with `content-primary` text and subtle border
  - Timestamp: `content-tertiary` with opacity for visual hierarchy
  
- ✅ **Accessibility (WCAG AAA)**
  - `role="article"` on message container
  - `aria-label` describing sender and content for screen readers
  - Proper semantic structure with time metadata
  
- ✅ **Mobile Responsive**
  - Max-width constraint prevents excessively wide bubbles
  - `break-words` ensures text wraps on narrow screens
  - Proper spacing and padding for touch targets

### TypingIndicator.tsx
- ✅ **Loading Animation**
  - Three bouncing dots with staggered animation delays
  - Uses standard CSS `animate-bounce` with custom animation delays
  - Responsive to reduced motion (uses Tailwind's prefers-reduced-motion respect)
  
- ✅ **Accessibility**
  - `role="status"` for live region announcements
  - `aria-label` describes the typing action
  
- ✅ **Styling**
  - Matches assistant message bubble styling for visual consistency
  - Uses `content-tertiary` color for subtle appearance

### ChatInput.tsx
- ✅ **Form Handling**
  - Auto-focusing textarea for immediate user interaction
  - Enter key sends message, Shift+Enter creates new line (standard chat UX)
  - Disabled state prevents submission when input is empty
  
- ✅ **Accessibility**
  - `aria-label` on input and button for screen readers
  - Proper focus ring styling using brand color token
  - Disabled state indicated visually with opacity
  
- ✅ **Mobile Optimization**
  - Uses `<textarea>` instead of `<input>` for multi-line support
  - `rows={1}` with `resize-none` for compact default, but allows growth
  - No hardcoded `dvh` yet (Phase 07 mobile optimization will handle virtual keyboard pinning)
  - Accessible through Tab navigation

## Technical Decisions & Rationale

### Color System
All components use CSS variable helpers from `@/lib/utils` (`colorVar()`, `bgColorVar()`, etc.) instead of hardcoded values. This maintains single source of truth in `theme.css` and ensures design token compliance across light/dark modes.

### Markdown Support Deferred
Phase 06 Batch 1 focused on UI structure. Markdown rendering for assistant messages will be added in Batch 2 when `react-markdown` is installed and integrated into the full `Asisten.tsx` page.

### State Management Not Yet Implemented
ChatInput and ChatBubble are pure presentational components. Conversation state management (Zustand or Context) will be implemented in Batch 2 when building the full Assistant page.

### Touch Target Sizes
- Input field: 44px minimum height (standard mobile touch target)
- Send button: 44px minimum height
- Message bubbles: Content-focused, text size 14px (readable on mobile)
