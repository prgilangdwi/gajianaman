## 4. ARCHITECTURE NOTES

### PDF/Charts chunks — NOT an initial load problem
`Laporan.tsx` and `Riwayat.tsx` (the only consumers of `html2pdf`/`recharts` via `vendor-pdf` and `vendor-charts`) are both wrapped in `React.lazy()` in `App.tsx`. Vite code-splits them into separate chunks. Lighthouse on `/home/overview` would NOT load these chunks — they only load when the user navigates to those pages. The chunk size warnings are about the lazy chunk sizes, not the initial load. This is acceptable behavior.

### CSP widening — `connect-src` *.supabase.co
The original CSP had `connect-src 'self' https://api.supabase.co`. Supabase uses multiple subdomains for different services:
- `https://[ref].supabase.co` — PostgREST (database)
- `https://[ref].auth.supabase.co` — Auth (OAuth flows)
- `https://[ref].realtime.supabase.co` — Realtime subscriptions
Widening to `https://*.supabase.co` ensures all Supabase services work without CSP violations.

### manifest.webmanifest icon strategy
Used `"sizes": "any"` with the existing `dark-logo.png`. Lighthouse accepts this for the installability check. A proper PWA would need multiple icon sizes (192×192, 512×512) — this is a functional minimal implementation, not a full PWA setup. Upgrading icon sizes is a Phase 6 candidate.

### Google Fonts `display=swap` already set
`fonts.css` uses the Google Fonts URL parameter `&display=swap`. This is equivalent to `font-display: swap` in `@font-face` declarations and is the recommended approach. No change needed.
