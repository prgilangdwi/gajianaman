## Technical Implementation Notes

### TransactionList.tsx
- **Responsive Breakpoint**: `useMediaQuery('(min-width: 768px)')` to switch layouts
- **Mobile (<768px)**:
  - Card-list layout with SwipeAction wrapper
  - Swipe left → delete callback with haptic confirmation
  - Swipe right → toggle expandable detail section (shows note + transaction ID)
  - AnimatePresence for smooth card exit on delete
  - Staggered animations via staggerChildren preset (50ms offset)
- **Desktop (≥768px)**:
  - Semantic `<Table>` component from shadcn/ui
  - Columns: Tanggal (date), Kategori (category), Keterangan (note), Jumlah (amount, right-aligned, monospace)
  - Motion animations with staggered row entry (idx * 0.03 delay)
  - Hover state: bg-slate-50 dark:bg-slate-800
- **Shared Features**:
  - Sorted by date descending (most recent first)
  - Loading state: skeleton placeholders (5 items)
  - Empty state: 📝 emoji + "Belum ada transaksi untuk periode ini"
  - Both layouts respond to useReducedMotion()
  - On-row click → onEdit callback (for future modal/drawer integration)

### Layout.tsx
- **Import Change**: Replaced `TransactionModal` with `QuickAddSheet`
- **State Management**: Changed `isTransactionModalOpen` → `isQuickAddOpen`
- **FAB Props**:
  - `isOpen={isQuickAddOpen}`
  - `onClose={() => setIsQuickAddOpen(false)}`
  - `type="expense"` (default; income type can be added as feature toggle)
- **FAB Positioning**: Maintained existing `bottom-20 right-4 md:bottom-8 md:right-6 z-30`
  - Mobile (default): 80px from bottom (5rem) = 20px clearance above BottomNav (typically 60-70px)
  - Tablet+: 32px from bottom (8 = 2rem)
  - z-30 ensures it sits above BottomNav (z-20)
- **Animation**: `whileHover={{ scale: 1.1 }}`, `whileTap={{ scale: 0.95 }}` (motion/react)
- **Accessibility**: `aria-label="Tambah transaksi cepat"` (changed from "Tambah transaksi")

### BudgetInlineEdit.tsx
- **Display Mode**:
  - Shows budget amount (monospace) + percentage (status-colored)
  - Progress bar (0-100%, clamped)
  - Spent vs budget label
  - Click to edit affordance ("Klik untuk edit" text)
- **Edit Mode** (triggered by click):
  - CurrencyInput from Batch 2 (numeric input with IDR formatting)
  - Confirm button (h-11 = 44px, green, Check icon)
  - Cancel button (h-11 = 44px, gray, X icon)
  - Live progress bar + percentage during edit
  - Keyboard support: Enter to confirm, Escape to cancel
- **Supabase Integration**:
  - Validates amount > 0 before saving
  - Upserts budget record with (user_id, category, month, year)
  - Persists: user_id, category, amount, period ('monthly'), month, year
  - Toast feedback: success message with formatted amount
- **Error Handling**:
  - Missing auth → "Anda harus login terlebih dahulu"
  - Amount ≤ 0 → "Jumlah budget harus lebih dari 0"
  - Save failure → "Gagal menyimpan budget"
- **Status Colors**:
  - Red (sentiment-negative): > 100%
  - Orange (sentiment-warning): 80-100%
  - Green (sentiment-positive): < 80%
- **Read-Only Mode**: `readOnly` prop disables editing for display-only contexts

## Architecture Notes

1. **Responsive Strategy**: Single codebase (no route-level responsive splits) using CSS media queries + React hooks (useMediaQuery). TransactionList conditionally renders card-list or table based on viewport.
2. **Gesture Integration**: SwipeAction from Batch 1 reused without modification; TransactionList wraps each card with SwipeAction for mobile gestures.
3. **State Management**: Local state in BudgetInlineEdit (edit mode toggle, temp amount); Supabase session for auth + user_id extraction.
4. **Animation System**: motion/react presets (fadeUp, staggerChildren) with GPU-accelerated transforms; no jank on 60fps scroll.
5. **Accessibility**: Semantic HTML on desktop (<Table>, <tr>, <td>); role="dialog" not needed (card-based, not modal); aria-labels on all buttons.
