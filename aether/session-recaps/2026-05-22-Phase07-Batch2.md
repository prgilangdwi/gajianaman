## Technical Implementation Notes

### CurrencyInput.tsx
- **Input Handling**: Raw digits only; `replace(/\D/g, '')` to extract numeric input
- **Display**: `Intl.NumberFormat('id-ID', { style: 'currency', currency: 'IDR' })` for Rp formatting
- **Internal State**: Stores integer value (e.g., 150000) while displaying "Rp 150.000"
- **Font**: DM Mono for data integrity alignment (Principle 11)
- **Touch Target**: min-h-[44px] with py-2.5 padding = 48px total height
- **Mobile UX**: `inputMode="numeric"` triggers native keyboard on mobile
- **Validation**: Max value clamped to 1 billion IDR (reasonable transaction limit)
- **Accessibility**: `aria-label="Jumlah transaksi dalam Rupiah"`

### QuickAddSheet.tsx
- **Form Flow**: Amount → Category → Note → Submit (frictionless 4-tap flow)
- **Bottom Sheet Integration**: Uses BottomSheet primitive from Batch 1 with custom snapPoints [0.5, 1]
- **Category Grid**: 3-column layout; all buttons 44px+ touch target; emoji + label per category
- **Async Submission**: 
  - Validates amount > 0 and category selected
  - Fetches user ID from `supabase.auth.getSession()`
  - Inserts transaction with user_id, amount, type, category, note, date
  - Shows toast feedback (error/success)
  - Resets form on successful insert
- **Keyboard Support**: Enter key submits form; Escape closes sheet (via BottomSheet)
- **Motion Integration**: `staggerChildren` animation on form fields (50ms offset per Principle 10)
- **Supabase Types**: Uses `Transaction` type from supabase.ts; proper type casting on insert response
