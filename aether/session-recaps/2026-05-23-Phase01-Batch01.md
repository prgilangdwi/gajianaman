## Decisions & Trade-offs

1. **ErrorBoundary Strategy:** Chose to wrap at route level (App.tsx) and page level (Laporan.tsx) rather than per-component granularity. This balances crash recovery with minimal performance overhead.

2. **Route Deletion:** Removed the broken `/split/:token` route entirely rather than redirecting it. This prevents users from accessing undefined functionality and surfacing the issue clearly via 404.

3. **Duplicate Imports:** Kept the first DropdownMenu import location (13-17 was removing, 34-38 was kept). Both were identical; removed the first to prevent shadowing.

4. **Amount Step:** Changed from 100 to 1 for recurring bills because Rupiah-based input should allow fine granularity (1 Rp increments are meaningful for Indonesian finance).
