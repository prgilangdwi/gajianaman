## Key Implementation Details

### chatStore.ts (Zustand Store)
- ✅ **State Management** (Principle 12: Minimal Context)
  - `messages: ChatMessage[]` — Full conversation history
  - `isLoading: boolean` — API call in progress
  - `error: ChatError | null` — Error state with message and code
  - Devtools middleware for debugging
  
- ✅ **Actions**
  - `addMessage()` — Append user/assistant message
  - `setLoading()` / `setError()` — Update UI states
  - `clearMessages()` — Reset conversation
  - `retryLastMessage()` — Error recovery (Principle 08: Error Resilience)
  
- ✅ **Selectors** (Performance optimization)
  - `useChatMessages()` — Messages only
  - `useChatLoading()` — Loading state only
  - `useChatError()` — Error state only
  - `useChatActions()` — All actions (prevents excessive re-renders)

### ChatBubble.tsx Enhanced (Markdown Rendering)
- ✅ **Markdown Parser** (No external dependencies)
  - Supports: **bold**, *italic*, `code`, [links](url)
  - Regex-based parsing with proper escaping
  - Renders as React components (safe HTML, proper sanitization)
  
- ✅ **Styling Integration**
  - Code blocks: `bg-neutral` background with mono font
  - Links: `content-link` color with underline and target="_blank"
  - All styles use CSS variables for theme consistency
  
- ✅ **Line-by-line Processing**
  - Splits by newline to preserve formatting
  - Each line parsed independently for markdown patterns
  - Maintains whitespace for intentional spacing

### Asisten.tsx Refactored (Multi-Turn Chat)
- ✅ **UI Architecture** (from Q&A card to chat interface)
  - Full-height flex layout with three sections:
    1. Header (title, description)
    2. Messages area (scroll container, auto-scroll to bottom)
    3. Input area (ChatInput component)
  
- ✅ **Message Management** (Principle 04: Progressive Disclosure)
  - Auto-scroll on new messages: useEffect with scrollIntoView
  - Messages render in order with timestamps
  - TypingIndicator shows during API calls
  - Error messages display inline with sentiment-negative color
  
- ✅ **Error Resilience** (Principle 08)
  - Try-catch wraps API call
  - Error displayed to user with recovery message
  - Error state prevents double-sends (isLoading guard)
  - Retry via quick prompts available during error
  
- ✅ **Conversation Context**
  - Conversation history passed to API
  - Each message includes timestamp for context
  - User and assistant messages tracked separately
  
- ✅ **Quick Prompts**
  - Display in empty state (when no messages)
  - Pre-populated suggestions
  - Disabled during loading (prevents duplicate sends)
  
- ✅ **Mobile Responsiveness**
  - Header stays fixed (flex-shrink-0)
  - Messages area flex-1 (grows to fill space)
  - Input stays pinned at bottom
  - No hardcoded widths; responsive to container
