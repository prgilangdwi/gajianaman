## Dark Mode Integration

### Hook Integration
- Settings component imports and uses `useDarkMode()` hook
- Reads `isDarkMode` state directly from hook
- Calls `toggleDarkMode()` function on button click
- Waits for `mounted` flag (SSR-safe) before rendering toggle

### Behavior
- When user clicks "On/Off" button, `toggleDarkMode()` is called
- Hook applies theme by adding/removing `.dark` class on `<html>`
- User preference saved to localStorage under key `gajian_aman_darkmode`
- Theme persists across page refreshes and sessions

### CSS Integration
- Uses `@media (prefers-color-scheme: dark)` + `.dark` class approach from Batch 1
- Settings UI respects current theme (colors automatically switch via CSS variables)
- No manual theme application needed in Settings component

## Language Switching

### localStorage Persistence
- Key: `gajian_aman_language`
- Values: `'id'` (Indonesian) or `'en'` (English)
- Default: `'id'` (Indonesian)

### Custom Event Dispatch
- Dispatches `languageChange` event when user selects language:
  ```typescript
  window.dispatchEvent(new CustomEvent('languageChange', { detail: { language: lang } }));
  ```
- Ready for future i18n integration (e.g., i18next, react-i18next)
- App components can listen to this event for global language switching

### Bilingual Translations Object
- All UI strings stored in `translations` object keyed by language
- Enables easy expansion to additional languages
- Used throughout Settings component for all labels, descriptions, and button text

## Technical Notes

### Dark Mode Hook Integration
- Settings component imports `useDarkMode()` from `@/hooks/useDarkMode`
- Hook returns: `{ isDarkMode, mounted, toggleDarkMode, setSystemPreference }`
- Component uses only `isDarkMode`, `mounted`, and `toggleDarkMode` (setSystemPreference reserved for future UX)
- SSR-safe: waits for `mounted` flag before rendering toggle

### Language Switching Pattern
- localStorage read on component mount
- Custom event dispatched to enable global listeners
- No auto-reload needed; listeners can update UI without page refresh
- Compatible with React Router navigation (Settings route accessible while app language changes)

### Navigation Pattern

- All links wrapped with ErrorBoundary and RequireAuth (inherited from Layout wrapper)
- No new navigation config needed (Settings not part of NAV_SECTIONS; accessed via direct Link)
