## 3. ARCHITECTURE NOTES

- **Fallback budget contract:** The API endpoint always returns `200` with a valid `BudgetRecommendation` shape — the hook's catch block returns `{ budgetItems: [] }` as the fallback when the fetch itself fails (network error, timeout). The wizard detects `budgetItems.length === 0` as the error signal and shows the retry toast. This means: API error → empty budgetItems → error toast. API success with fallback data → non-empty budgetItems → navigate to confirm. Two separate failure modes are handled consistently.
- **`handleSubmit` self-reference in toast action:** The `toast.error` `action.onClick` references `handleSubmit` itself for the retry. This works because `handleSubmit` is declared with `useCallback` and by the time the toast action is clicked, the callback reference is stable in the closure. No circular dependency issue.
- **`discretionary` in fallback:** The fallback budget totals to `discretionary * 1.0` (100%) across 7 categories, which satisfies the `totalRecommended = discretionary` contract even without Claude.
- **Route `/gajian/confirm` not yet wired:** Navigate will fail gracefully in the browser (404 or redirect to login). This is expected and acceptable — Batch 3 wires the route.
