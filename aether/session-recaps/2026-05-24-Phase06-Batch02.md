## 4. CRITICAL FINDING — React.lazy() False Positives

**The 150-file deletion target from the staging doc is unachievable without breaking the app.**

react-doctor v0.2.3 marks files as "unused" using static import-graph traversal. It cannot resolve `React.lazy(() => import('./pages/Overview'))` patterns — these are dynamic imports evaluated at runtime. The result:

- Every page component (`Overview.tsx`, `Budget.tsx`, `Riwayat.tsx`, etc.) is flagged as unused
- Every hook/component imported by those pages is transitively flagged
- The entire app tree (App.tsx, Layout.tsx, all hooks, all pages) appears as "unused"
- Only the true entry point `main.tsx` is recognized as the root

**Real unused vs. false positive breakdown (after this batch):**
- Total flagged: 184 remaining
- Genuine orphans remaining: 0 (all confirmed orphans are now deleted)
- False positives: ~184 (entire lazy-loaded app tree)

**Score impact:** Deleting 20 genuine orphans moved score from 57 → 58 (+1). The estimated +15 in the staging doc assumed 150+ real deletions; those 150 files are actually the live app.

**Implications for Batch 3:**
The score gap to reach ≥80 must be closed through warning fixes (Batch 3), not through further dead-code deletion. The remaining 184 "unused-file" flags are permanent false positives for this react-doctor version; they cannot be resolved without either:
a) Upgrading react-doctor to a version that understands React.lazy()
b) Removing React.lazy() routing (would break code-splitting)
c) Accepting the scoring penalty
