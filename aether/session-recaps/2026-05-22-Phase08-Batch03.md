## 3. ARCHITECTURE NOTES
- **Expandable Row Pattern:** Click header to toggle details panel; animated collapse/expand via Motion library. Collapsed state shows date (short), category chip, truncated tags (shows +N indicator), amount. Expanded state shows full date (long), wallet name, complete tags, subcategory, AI confidence percentage, full note.
- **Sort Implementation:** useMemo hook applies sort logic (date ascending/descending, amount ascending/descending) after filtering; dropdown button with checkmarks indicates active sort. Default: date descending (newest first).
- **Color Picker UI:** 16-color palette grid + hex input field; preview circle shows selected color; selected color shows ring indicator.
- **Category Management:** Three sections (Pengeluaran/Pemasukan/Simpanan) with category cards showing icon, name, color preview, custom vs default badge, delete button (disabled for defaults).
- **Merge Workflow:** Two-step select (source category, then target category with source filtered out); confirmation shows transaction count message; disabled state until both selections made.
- **Design Token Consistency:** All components use bgColorVar, textColorVar, borderColorVar helpers; ≥44px touch targets (h-9 sm:h-10 for buttons, h-10 for merge/sort buttons).
- **Motion & Animations:** ExpandableTransactionRow uses smooth height/opacity transition (200ms); category cards use layout animation with scale; sort/merge dialogs use standard Motion fade.
- **Responsive Layout:** Categories page uses grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 for category cards; sort button hides label on mobile (icon only), shows on desktop.
