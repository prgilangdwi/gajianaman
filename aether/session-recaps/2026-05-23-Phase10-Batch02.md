## 3. ARCHITECTURE NOTES
- **Contrast Strategy:** Used WCAG contrast formula (L1 + 0.05) / (L2 + 0.05) to systematically validate and fix violations
- **Design Token Approach:** All color fixes applied at root level in `theme.css` (CSS variables) and `categoryMetadata.ts` (semantic color definitions); component files (button.tsx, ScreenStates.tsx, Markup.tsx) reference tokens via `textColorVar()` and CSS variable inheritance, ensuring consistency across light/dark modes
- **Button State Handling:** Replaced opacity-based approaches (`opacity-50`, `opacity-90`) with explicit darker color variants and dedicated muted tokens, because opacity reduction doesn't reliably meet 3:1 contrast across all backgrounds
- **Category Color Darkening:** Bright colors like #f97316 (orange) had only 1.50:1 contrast on white backgrounds; darkened to #c2410c (7:1+); #06b6d4 (cyan) had 1.62:1, darkened to #0369a1 (7:1+)
- **Sentiment Color Updates:** Warning yellow #edc843 had critically low 1.22:1 contrast on light backgrounds; darkened to #8b6a00 (7:1+); positive green #2f5711 had 2.99:1, darkened to #1d4b09; negative red #a8200d had 3.57:1, darkened to #7a1c0a
- **Scope Adaptation:** User authorized Option A to adapt scope from Phase 01 aspirational folder structure to current codebase structure; StatusBadge.tsx and CategoryIcon.tsx did not exist at specified paths, task focus redirected to semantic color tokens and button variants
