## DETAILED IMPLEMENTATION

### 1. TrendBadge.tsx

**Purpose:** Display trend direction with automatic sentiment coloring.

**Features:**
- Three trend states: `'up'` (positive), `'down'` (negative), `'stable'` (neutral)
- Auto-selected icons: TrendingUp, TrendingDown, Minus from lucide-react
- Optional value display (e.g., "+15%", "-8%")
- Sentiment-based CVA styling (green/red/neutral backgrounds + text colors)
- Accessibility: `role="status"`, `aria-label` with trend + value

**Props Interface:**
```typescript
interface TrendBadgeProps extends React.HTMLAttributes<HTMLDivElement> {
  value?: number                              // Percent value to display
  trend: 'up' | 'down' | 'stable'             // Direction indicator
  label?: string                              // Optional label text
  showIcon?: boolean                          // Show/hide trend icon (default: true)
  sentiment?: 'positive' | 'negative' | 'neutral'  // Override auto sentiment
}
```

**Token Usage:**
- `--color-sentiment-positive-bg`, `--color-sentiment-positive` (green)
- `--color-sentiment-negative-bg`, `--color-sentiment-negative` (red)
- `--color-bg-neutral`, `--color-content-secondary` (neutral)
- `--radius-sm` (8px border radius)

### 2. ProgressBar.tsx

**Purpose:** Semantic linear progress bar with status-based coloring.

**Features:**
- Four status states: `'safe'` (≤70%), `'warning'` (70-100%), `'over'` (>100%), `'none'` (unset)
- Auto-calculated status from progress value if not explicitly set
- Optional label above bar
- Optional percentage text
- Responsive width via `style={{ width: 'X%' }}`
- Accessibility: `role="progressbar"`, `aria-valuenow`, `aria-valuemin/max`

**Props Interface:**
```typescript
interface ProgressBarProps extends React.HTMLAttributes<HTMLDivElement> {
  progress: number                            // 0-100 (auto-clamped)
  status?: 'safe' | 'warning' | 'over' | 'none'  // Budget status
  label?: string                              // Optional label
  showPercentage?: boolean                    // Show % text (default: false)
  showLabel?: boolean                         // Show label (default: true)
}
```

**Token Usage:**
- `--color-sentiment-positive-bg`, `--color-sentiment-positive` (safe)
- `--color-sentiment-warning-bg`, `--color-sentiment-warning` (warning)
- `--color-sentiment-negative-bg`, `--color-sentiment-negative` (over)
- `--color-bg-neutral` (none/background)
- `--space-2`, `--space-3` (spacing)
- `--radius-full` (pill shape)

### 3. ComparisonRow.tsx

**Purpose:** Side-by-side comparison of two metrics with optional icons/labels.

**Features:**
- Flexible layout: row (default) or column orientation
- Left/right metric display with optional icons and labels
- Optional comparison text (e.g., "vs. budget", "vs. last month")
- Semantic background for column layout (neutral bg with padding)
- Monospace typography for values (DM Mono font)
- Accessibility: `role="region"`, `aria-label`

**Props Interface:**
```typescript
interface ComparisonRowProps extends React.HTMLAttributes<HTMLDivElement> {
  label: string                               // Section title
  value1: React.ReactNode                     // Left value
  value2: React.ReactNode                     // Right value
  label1?: string                             // Left label (e.g., "Budget")
  label2?: string                             // Right label (e.g., "Spent")
  icon1?: React.ReactNode                     // Left emoji/icon
  icon2?: React.ReactNode                     // Right emoji/icon
  comparison?: React.ReactNode                // Comparison text
  direction?: 'row' | 'column'                // Layout (default: 'row')
}
```

**Token Usage:**
- `--text-body`, `--text-caption` (typography)
- `--text-body-lg` (value size)
- `--color-content-primary`, `--color-content-secondary`, `--color-content-tertiary`
- `--space-2`, `--space-3`, `--space-4` (spacing)
- `--radius-md` (card corners in column mode)
- `--color-bg-neutral` (background in column mode)

### 4. MetricCard.tsx

**Purpose:** Analytical metric card for dashboard and report views.

**Features:**
- Six variant styles: `'default'`, `'elevated'`, `'success'`, `'warning'`, `'error'`, `'neutral'`
- Icon display (emoji or SVG)
- Optional subtext and trend indicator
- Optional clickable behavior with keyboard support (Enter/Space)
- Text color automatically determined by variant (overridable)
- Elevation shadows via Tailwind (shadow-card, shadow-card-hover, shadow-float)
- Accessibility: `role="button"` when clickable, `tabindex=0`, keyboard handlers

**Props Interface:**
```typescript
interface MetricCardProps extends React.HTMLAttributes<HTMLDivElement> {
  label: string                               // Uppercase label
  value: React.ReactNode                      // Main value (mono)
  icon?: React.ReactNode                      // Optional emoji/icon
  subtext?: React.ReactNode                   // Small text below value
  trend?: React.ReactNode                     // Trend badge component
  variant?: 'default' | 'elevated' | 'success' | 'warning' | 'error' | 'neutral'
  interactive?: boolean                       // Enable click styling
  textColor?: string                          // Override text color
  onClick?: () => void                        // Click handler
}
```

**Token Usage:**
- `--color-bg-card`, `--color-bg-elevated` (backgrounds)
- `--color-sentiment-*` for colored variants
- `--shadow-card`, `--shadow-card-hover`, `--shadow-float` (elevations)
- `--radius-lg` (16px border radius)
- `--space-3`, `--space-4` (internal padding)
- `--text-h2`, `--text-body`, `--text-caption` (typography)

## ARCHITECTURE DECISIONS

### Decision 1: Separate Components vs. Unified Visualization Component
**Choice:** Separate, granular components (TrendBadge, ProgressBar, ComparisonRow, MetricCard)  
**Rationale:** Composability. Each primitive solves ONE visualization need. Laporan/Tren/Forecasting can mix-and-match based on their requirements.  
**Tradeoff:** 4 imports vs. 1, but flexibility and reuse are maximized.

### Decision 2: CVA Variant Pattern for Styling
**Choice:** class-variance-authority (cva) with CSS variables  
**Rationale:** Matches existing codebase (BadgeBase, etc.). Type-safe variants. Tailwind integration.  
**Tradeoff:** ~5 more lines per component for CVA setup, but eliminates inline styles.

### Decision 3: Auto-Sentiment Detection for TrendBadge
**Choice:** Trend ('up'/'down'/'stable') → auto-determine sentiment (positive/negative/neutral)  
**Rationale:** Reduces prop burden on consumers. Sentiment is deterministic from trend.  
**Tradeoff:** Can override via explicit sentiment prop if needed (flexibility preserved).

### Decision 4: SVG Icons in TrendBadge (Lucide React)
**Choice:** Use lucide-react icons (TrendingUp, TrendingDown, Minus)  
**Rationale:** Accessible SVG icons. Already in project dependencies. Consistent with icon patterns.  
**Tradeoff:** Small bundle impact, but negligible (~1KB gzip across all icons).

### Decision 5: No Animation Logic in Primitives
**Choice:** Defer animations to consumers (Batch 2 components)  
**Rationale:** Primitives are presentation-only. Animations belong to high-level components.  
**Tradeoff:** Consumers must add `motion/react` wrappers if animation needed (slight duplication).
