## 3. ARCHITECTURE NOTES
- **Inline Editing Pattern:** BudgetRow implements focus-based editing (click to edit, save on blur or Enter key). No modal overhead; changes persist immediately to Supabase via upsertBudget.
- **Milestone Visualization:** GoalMilestone uses 4 visual indicators (progress bars) to show milestone achievement (25%/50%/75%/100%), providing granular progress feedback.
- **Contribution Flow:** Direct input field in GoalMilestone allows users to add funds to goal without modal friction; updates saved_amount directly via updateGoalSaved.
- **Countdown Logic:** Deadline deadline rendering uses date-fns with Indonesian locale (idLocale) to show relative time ("X hari lagi", "Hari ini", "Tenggat terlewati").
- **Design Token Consistency:** Both new components use bgColorVar, textColorVar, borderColorVar helpers and maintain ≥44px touch targets (h-10 buttons/inputs per Phase 07).
- **Error Handling:** Inline saves show toast feedback on error; contribution flow validates input before submitting to API.
