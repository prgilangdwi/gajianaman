## 3. ARCHITECTURE NOTES

- **Mobile sticky buttons:** `BudgetConfirmation.tsx` uses `fixed bottom-0` for the action button row on mobile with `sm:static` override on tablet+. The page body has `pb-28 sm:pb-8` to prevent content being obscured behind the fixed bar. This matches the mobile UX pattern for confirmation pages.
- **Separate edit toggle in header vs footer:** The "Ubah Sendiri" button appears both in the card header (inline, text-only) and in the sticky footer (outline variant). Both call `setIsEditMode((v) => !v)`. This is intentional — the header button is convenient when the user sees the list; the footer button is always accessible.
- **`gajian_setup_complete` vs `onboarded`:** The executor prompt correctly identified the column discrepancy. The staging doc referenced `onboarded` (wrong); the actual `supabase.ts` User interface at line 102 has `gajian_setup_complete?: boolean`. All code uses `gajian_setup_complete`.
- **DetailKategori dormant bug:** Phase 3 created the file but never wired the route. The `react-router-dom` import was never resolved at build time. Batch 3's route wiring caused the module to be included in the build graph, surfacing the error. Fix is minimal and correct.
- **`handleSave` error handling:** On Supabase error, `toast.error` is shown but navigation does NOT occur. The user remains on `/gajian/confirm` and can retry. This matches the spec.
