## 3. ARCHITECTURE NOTES

### Principle 02 (Component Atomicity) Applied
- CardBase: Single responsibility — apply shadow elevation and padding to card-like containers
- BadgeBase: Single responsibility — display status/category labels with semantic colors
- ChipBase: Single responsibility — interactive filter toggle with close affordance
- All components are composable, reusable atoms with no assumptions about parent context

### Principle 03 (Token-Driven Styling) Applied
- CardBase: All colors from CSS variables (--color-bg-card, --radius-md, --duration-fast, --shadow-*)
- BadgeBase: All sentiment colors from CSS variables (--color-sentiment-*), neutral from --color-bg-neutral and --color-border-neutral
- ChipBase: Active state uses --color-brand-primary, inactive uses --color-bg-neutral + --color-border-neutral
- Zero hardcoded hex values across all three components

### Shadow System Integration
CardBase leverages the 6-level shadow system established in Phase 02 Batch 1:
- `surface` variant → --shadow-card (1px elevation, resting)
- `elevated` variant → --shadow-card-hover (4px elevation, interactive)
- `floating` variant → --shadow-float (8px elevation, prominent)
- Enables consistent elevation hierarchy across all card-like components

### Category Color Styling Strategy
BadgeBase uses `color-mix()` CSS function to dynamically blend category colors:
- Syntax: `color-mix(in srgb, ${categoryColor} 15%, transparent)`
- 15% saturation creates appropriate background tint while maintaining color identity
- Text uses full saturation for contrast
- Benefits: No need to pre-compute background variants for 10 category colors; scales automatically if categories added

### Accessibility Considerations
- BadgeBase: Sentiment colors chosen for AAA contrast compliance per Phase 02 spec
- ChipBase: Focus ring matches ButtonBase pattern (ring-[3px]), keyboard navigation via native button
- ChipBase close button: Separate click handler with hover feedback (hover:bg-current/10), no form submission interference
- All components support disabled state with opacity + cursor management
